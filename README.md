# 苍梵界跑团房规


`cfj-sanity-system` 是苍梵界 DND5e 跑团使用的房规模块。当前版本主要处理理智规则：SAN 检定、SAN 豁免、理智损失、理智状态、裂解/崩溃症状、短休和长休恢复。模块也包含默认关闭的持续伤势房规，以及内置的骑乘、下马和跟随 Token 工具。理智、持续伤势和骑乘在模块设置中各有独立分区，并且都可以单独关闭。

模块不再要求把多个宏按钮摆在外面。GM 的聊天输入区附近会出现 **跑团房规** 按钮；玩家端不会显示这个按钮。若按钮被其他聊天栏或骰子栏模块遮挡，可临时在聊天框输入 `/理智` 打开同一个控制台。GM 点开后的控制台、DC、同源、熟练、主动深入和目标选择都只对 GM 可见。DM 发起后，在线目标玩家会收到强制判定弹窗和自己的判定卡；离线玩家不会生成本次 pending 请求。

## 安装地址

在 Foundry 的模块安装器中填写：

```text
https://raw.githubusercontent.com/knightinrain/cfj-sanity-system/main/module.json
```

## 安装步骤

1. 打开 Foundry VTT 管理后台。
2. 进入 **Add-on Modules / 附加模块**。
3. 点击 **Install Module / 安装模块**。
4. 粘贴上面的 Manifest URL。
5. 安装模块。
6. 进入世界，在 **Manage Modules / 管理模块** 中启用 **苍梵界跑团房规**。
7. 刷新一次世界页面。

## 日常使用

### DM

1. 在聊天输入区附近点击 **跑团房规**。如果按钮暂时没出现，在聊天框输入 `/理智`。
2. 聊天区会出现只对 GM 可见的 **苍梵界跑团房规控制台**。
3. 点击 **发起理智判定**。
4. 设置类型、DC、同源、是否加入熟练、是否主动深入，并勾选目标角色。
5. 发送后，每个目标角色必须有在线的非 GM 拥有者。在线目标玩家会收到强制判定弹窗和自己的判定卡；GM 也能看到这些卡。没有在线玩家拥有者的角色会被跳过。

`/理智`、`/san`、`/sanity` 仍保留为 GM 备用入口，但不再是主要操作方式。

### 玩家

1. 等 DM 发起理智判定。
2. 收到强制判定弹窗后，点击 **进行理智豁免** 或 **进行理智检定**。第一个弹窗必须完成掷骰才能关闭。
3. 不需要设置 DC，也看不到 DC 设置项。掷骰完成后，会收到一次性结果弹窗，可以手动关闭。
4. 也可以点角色卡上的 SAN 检定 / SAN 豁免，效果相同。

### 建卡或旧卡同步

1. GM 选中角色 token。
2. 点击聊天区附近的 **跑团房规**。如果按钮暂时没出现，在聊天框输入 `/理智`。
3. 点击 **初始化或刷新选中角色**。
4. 选择：
   - **生成新理智值**：按 `4d6kh3` 生成最大理智和当前理智。
   - **只连接/刷新**：不重掷，只同步当前理智、最大理智、状态和资源栏。

## 推荐设置

- `默认 DC`：15
- `玩家必须等待 GM 发起`：开启
- `理智显示资源栏`：主资源栏；如果角色已有用途，改用副资源栏、第三资源栏或不写入资源栏
- `短休自动处理理智`：开启
- `长休自动处理理智`：开启
- `自动生成裂解/崩溃症状`：开启
- `显示 GM 明细`：开启

## 理智损失

理智豁免失败后才降低当前理智。

| 结果 | 当前理智变化 |
| --- | --- |
| 成功 | 不降低 |
| 失败 | -1 |
| 失败 5 点或更多 | -2 |
| 失败 10 点或更多，或自然 1 | -3 |
| 主动深入 | 在上述基础上额外 -1 |

自然 20 成功。自然 1 按最强失败损失处理。

## 理智状态

理智状态按比例判断，不再按固定损失点数判断。

已损失理智比例 = （最大理智 - 当前理智） / 最大理智。

这样可以避免低最大理智角色一建卡就进入异常状态。只要当前理智等于最大理智，角色就是稳定。

| 已损失理智比例 | 状态 | 效果 |
| --- | --- | --- |
| 0%-15% | 稳定 | 无影响 |
| 16%-34% | 动摇 | 下一次同源相关检定或豁免的结果减去 1d4 |
| 35%-54% | 失衡 | 对同源理智豁免具有劣势；不能从同源现象获得优势 |
| 55%-79% | 裂解 | 进入该状态时获得 1 个裂解症状 |
| 80%+ 或当前理智为 0 | 崩溃 | 进入该状态时获得 1 个崩溃症状 |

动摇和失衡只使用状态效果。只有裂解和崩溃会生成症状。角色卡效应、聊天结果和玩家结果弹窗都会显示当前理智、已损失比例和理智状态。

## 可选：持续伤势房规

持续伤势默认关闭。需要使用时，GM 在模块设置中开启 **启用持续伤势房规**。

这个规则只用于严重创伤，不替代生命值、死亡豁免或普通状态。当前版本不接管生命值伤害结算，不自动修改生命值、生命值上限或装备。模块会把随机表结果做成 ActiveEffect；若开启【持续伤势】自动套用明确机械效果，已规则化的伤势会同时写入 DAE/Midi 可读取字段。无法可靠识别的结果只做记录，不硬套惩罚。

### 触发条件

满足以下条件时，GM 可以要求投掷一次持续伤势表：

| 触发 | 使用条件 |
| --- | --- |
| 暴击 | 攻击暴击，并且该次攻击造成实际伤害。若伤害被临时生命值或免疫完全抵消，通常不触发。 |
| 豁免严重失败 | 角色在会造成伤害的豁免中失败，且失败值达到模块设置的严重失败差值。默认是失败 5 点或更多。 |
| GM 手动裁定 | 陷阱、坠落、仪式反噬、污染爆发、神性现象或剧情创伤足够严重，且普通生命值损失不足以表现后果。 |

同一次攻击、法术、陷阱或环境事件通常只投一次。若一次伤害包含多种伤害类型，使用造成伤害最高的类型；无法判定时，由 GM 选择最贴合叙事的一种。

### 不触发

以下情况通常不触发持续伤势：

- 未造成实际伤害。
- 普通小战斗中的轻微命中。
- 纯叙事擦伤、疲劳或尴尬后果。
- 玩家不在当前跑团场次。
- GM 判断会明显打断节奏，或该场景已经由理智、恐惧、疾病、诅咒等规则承担主要后果。

### 使用方式

1. GM 打开聊天输入区附近的 **跑团房规**；如果按钮暂时没出现，在聊天框输入 `/理智`。
2. 点击 **持续伤势规则** 查看当前开关、触发条件和使用限制。
3. 若已启用持续伤势，点击 **投掷持续伤势**。
4. 选择触发原因和伤害类型。
5. 模块投掷 `3d6`，自动读取对应随机表，并生成只对 GM 可见的持续伤势结果。
6. 若开启 **【持续伤势】自动添加伤势效应**，模块会把结果写入目标角色的 ActiveEffect，玩家和 GM 可在角色效应页查看。
7. 聊天卡会显示本次伤势被规则化成哪一类，以及对应自动效果。
8. 若开启 **【持续伤势】自动套用明确机械效果**，模块会根据结果文字规则化伤势，并写入可被 DAE/Midi 读取的 effect changes。关闭时，只记录伤势说明，不写入机械惩罚。

若结果要求确定部位，GM 再根据场景选择大肢体、小肢体表，或直接指定最合理的部位。

### 已规则化的自动效果

| 识别结果 | 自动效果 | 保留给 GM 的判断 |
| --- | --- | --- |
| 失去肢体 | 移动速度 -10 尺；所有攻击劣势；敏捷和力量检定劣势 | 若只影响单手或单脚，GM 可手动放宽。 |
| 眼部伤势 | 远程武器攻击劣势；远程法术攻击劣势；察觉检定劣势 | 完全失明仍由 GM 额外添加“目盲”。 |
| 骨折或行动伤 | 步行、攀爬、游泳速度 -10 尺；敏捷和力量检定劣势 | 上肢骨折可手动移除移动惩罚。 |
| 内伤 | 体质豁免劣势；专注豁免劣势 | 持续伤害、治疗 DC、是否恶化由 GM 处理。 |
| 失血 | 体质豁免劣势；专注豁免劣势 | 不自动每回合扣血，避免未实测 OverTime 误触发。 |
| 灼伤、腐蚀、冻伤、坏死、感染、中毒等持续性创伤 | 体质豁免劣势 | 是否持续扣血或恶化由 GM 处理。 |
| 疤痕或毁容 | 不自动惩罚战斗 | 社交影响按场景处理。 |

## 内置：骑乘房规

从 `0.2.0` 开始，骑乘功能已经并入 **苍梵界跑团房规**。不需要再单独安装或启用 `rideable_simple`。如果两个模块同时启用，可能出现重复 HUD 按钮或重复同步，建议只保留本模块。

### 使用方式

- 选择骑手 Token，目标指定一个坐骑 Token，然后按 `M`，或打开坐骑 Token 的 HUD 点击骑乘按钮。
- 选择已经骑乘的 Token 后按 `N`，或打开 Token HUD 点击下马按钮。
- 坐骑移动时，骑手会跟随坐骑位置同步。
- 下马时，骑手会落到坐骑旁边一格附近，而不是留在坐骑中心。
- 默认会给骑手添加可见的 **骑乘** 效应；擒抱模式会添加 **骑乘：被擒抱** 效应。

### 推荐设置

- **骑乘：默认所有 Token 可作为坐骑**：开启。
- **骑乘：强制体型限制**：开启。
- **骑乘：坐骑至少大几个体型**：1。
- **骑乘：体型规则模式**：至少达到体型差。
- **骑乘：每个坐骑最多骑手数**：2。
- **骑乘：骑手主动移动时**：记录为自由偏移，或按当前战斗需要改为拉回坐骑位置。

这些设置在 Foundry 的模块设置中，和理智、伤势设置放在同一个 **苍梵界跑团房规** 模块下面。
## 休息规则

- 短休：移除裂解症状，然后重新计算理智状态。
- 长休：恢复 1 点当前理智，移除理智症状，然后重新计算理智状态。
- 当前理智不会超过最大理智。
- 休息后的重新计算不会因为角色仍在裂解或崩溃而立刻再生成一个新症状。

## 验证清单

启用模块后，用测试角色验证，验证后删除测试角色：

- GM 端聊天输入区附近出现 **跑团房规** 按钮；若暂时不出现，聊天框输入 `/理智` 可以打开同一控制台。
- 玩家端不显示这个按钮。
- GM 点击按钮后，控制台聊天卡只对 GM 可见。
- 场景工具栏里不再出现单独的理智按钮。
- GM 发起一个非默认 DC 的理智判定。
- 目标玩家收到自己的判定卡，并能点击卡内按钮掷骰。
- 非目标玩家看不到该判定卡。
- 玩家掷骰时不能编辑 DC。
- 失败后当前理智降低，角色卡 SAN 数值变化。
- 进入裂解或崩溃时，角色获得可见状态和症状效应。
- 短休移除裂解症状。
- 长休恢复 1 点当前理智并移除理智症状。
